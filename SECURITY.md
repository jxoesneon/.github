# Security Policy

## Supported Versions

Please check individual repositories for specific version support. Generally, we support the latest major release.

## Reporting a Vulnerability

We take the security of our software seriously. If you have found a security vulnerability in any of our projects, please report it to us as described below.

**Please do NOT report security vulnerabilities in public GitHub issues.**

### How to Report

1.  Send an email to **[rj.joseeduardo@gmail.com](mailto:rj.joseeduardo@gmail.com)**.
2.  In the subject line, strictly format it as: `SECURITY ALERT: [Project Name] - [Vulnerability Type]`.
3.  Include details about the vulnerability, steps to reproduce, and any proof-of-concept code.

### Response

We will acknowledge receipt of your report within 48 hours and will strive to send you a detailed response within 5 days indicating the next steps.

### Attribution

If you wish, we will publicly acknowledge your responsible disclosure in the release notes of the patch that fixes the vulnerability.
